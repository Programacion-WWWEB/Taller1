Nombre del grupo

MusicaList

Descripción

Ud deberá desarrollar una aplicación web donde hayan dos tipo de usuarios diferentes (1) usuario administrador del sistema quien principalmente va crear, editar y eliminar los generos y sus canciones; (2) usuario votante donde el principal objetivo de esta es poder hacer listados de canciones ordenadas de la más votada a la menos quien además de crear y activar su cuenta va poder buscar canciones, verificar listas por género musical y finalmente votar y eliminar su voto, para ver el detalle de este partícular ud podrá verlos en la página web de historias de usuario.

Requerimientos

Funcionales

Historias de Usuarios

Usuario votante

HUV.1. Creación de cuenta
HUV.2. Activación de cuenta
HUV.3. Autenticación de usuario
HUV.4. Ver listas por género musical creadas
HUV.5. Buscar canción una canción
HUV.6. Votar/Desvotar por una lista

Usuario administrador

HUV.1. Autenticación de usuario
HUV.2. CRUD Géneros músicales
HUV.3. CRUD Canciones

No Funcionales

  • Para la conexión con la base de datos se deberá usar JPA.
  • Debe ser capaz de funcionar en sistemas operativos tanto Windows como macOS.
  • Las consultas a la base de datos deben ser eficientes y optimizadas para evitar tiempos de carga prolongados.
  • Debe ser intuitiva y fácil de usar, con un tiempo de aprendizaje mínimo para los usuarios.
  • La página debe ser capaz de manejar al menos 10 usuarios concurrentes sin degradar el rendimiento.
